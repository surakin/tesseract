# Emoji Shortcode Expansion — Design

## Context
Users type `:smile:` or `:gri` in the compose bar to get emoji inserted.
Codebase has ~1902 Unicode emoji entries (CLDR names, no shortcodes yet).
No autocomplete widget exists.

## Scope
Unicode emoji (`:smile:` → 😄) + custom MSC2545 emoticons (`:shortcode:` text).
Auto-expand on delimiter AND popup-while-typing (Option C from brainstorm).
HTML send-path for custom emoticons is Step 9 — out of scope here.

## Approach: Shared engine + shared popup widget (Approach B)
- `ShortcodeEngine` in `ui/shared/views/` — stateless, fully testable.
- `ShortcodePopup : tk::Widget` — shared widget, platform popup window.
- `NativeTextArea` gains `cursor_rect()`, `replace_range()`, `set_on_popup_nav()`.
- Each shell coordinates text monitoring, popup show/hide, key nav.

## Shortcode data
- `emoji::Entry` gains `shortcodes` field (space-separated, canonical first).
- Canonical = CLDR name → snake_case (generated by `emoji_data.gen.py`).
- `emoji_aliases.inc` adds common GitHub/gemoji aliases (MIT-licensed).
- Custom emoticons sourced from `client_->list_pack_images()` at query time.

## UX
- Popup appears above cursor when user types `:[a-z_]{2,}` (min 2 chars after `:`).
- Auto-expand: `:word:` + Space/Tab/EOT → replaced with glyph inline.
- Enter while popup row selected → accept (guard against submit).
- Popup hidden on: Escape, room switch, focus loss, compose clear.
- Custom emoticons: insert `:shortcode:` text; send-path HTML is Step 9.

## Popup row layout
Fixed width 280px. Each row: 28×28 cell (glyph or thumbnail) + `:shortcode:` label.
Max 8 rows visible.

## Media loading
Custom emoticon thumbnails use the existing `ensure_media_image_()` pipeline
(now backed by `MediaDiskCache`). Shell calls `repaint_shortcode_popup_()` when
a relevant `on_media_bytes_ready_` fires while popup is open.
